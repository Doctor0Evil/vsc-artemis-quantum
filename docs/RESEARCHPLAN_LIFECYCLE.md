# ResearchPlanShard Lifecycle

1. **Proposal**: AI-chat or researcher generates ResearchPlanShard
2. **Validation**: ALN schema + corridor impact analysis
3. **KER Scoring**: Compute expected K/E/R deltas
4. **Governance Lane**: Route to QVM-LOGIC-RESILIENCE, ECONET-KER-CORRIDOR, or IDENTITY-NEURORIGHTS
5. **Decision**: Accept (→ experiment), Reject (→ revision), or Defer (→ evidence gap)
6. **Implementation**: CorridorBand update as new shard (never in-place edit)
7. **Verification**: Phoenix pilot telemetry validates expected deltas
